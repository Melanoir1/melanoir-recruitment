## PART A. Verbal Identity — "The Measured Voice"

### A-1. Tone of Voice 4원칙

| 원칙 | 정의 |
|---|---|
| **Data-First** | 수치가 먼저 말하고, 철학이 뒤를 잇는다. 숫자로 약속한다. |
| **Subtraction, not Addition** | 더하지 않는다. 본질만 남긴다. 언어도 마찬가지다. |
| **Declaration, not Provocation** | 선동하지 않는다. 선언한다. 의문문보다 선언문. |
| **Non-competitive Confidence** | 경쟁을 선언하지 않는다. 새로운 범주를 정의한다. |

---

### A-2. 서술 7원칙

이 원칙은 모든 멜라누아 커뮤니케이션에 보편적으로 적용된다.
**원칙의 본질과 의도는 고정된다.** 채널·콘텐츠 유형별 구현 방식(표현 형식, 허용 범위 등)은 각 운영 문서에서 정의하며, 이는 원칙의 예외가 아니라 맥락에 맞는 실현이다.

**1. 오해를 먼저 소환한다**
독자가 이미 갖고 있는 불완전한 인식을 첫 단락에서 환기한다. 교정은 그 다음이다.

**2. 수치가 문장 앞에 온다**
"97%다. 이것은 X를 의미한다." — "X를 의미하는 수치인 97%다."가 아니다.

**3. 효과가 아닌 메커니즘을 설명한다**
"광보호 효과가 있다" → "흡수된 에너지를 열로 전환하는 Internal Conversion 반응이 피코초 단위로 일어난다."

**4. 비교 우위를 직접 주장하지 않는다**
기존 소재의 사실(IARC 분류, 잔류 기간 등)을 수치로 나열하고, 판단은 독자에게 남긴다.

**5. 법적 정의 없는 단어는 쓰지 않는다**
"안전하다", "뛰어나다", "효과적이다" 대신 "면역 반응 없음", "비자극 범주의 최솟값", "분해 경로 확인"으로 말한다.

**6. 모든 수치는 출처를 첨부한다**
DOI 또는 공인 표준 번호(ISO, IARC, REACH 등) 필수. 미확인 수치는 ⚠️ 표시 후 보수적 표현으로 조정.

**7. 종결 문장은 선언형으로 닫는다**
의문문, 청유형 금지. 마지막 문장은 항상 "~다."로 끝난다.

---

### A-3. 금지 단어 / 권장 단어

| 금지 | 권장 대체 |
|---|---|
| 좋다, 뛰어나다, 효과적이다 | 측정, 수치, 경로, 메커니즘 |
| 특별하다, 다양하다 | 설계, 기준, 검증 |
| 안전하다 (단독) | 면역 반응 없음 / 비자극 범주의 최솟값 |
| 무독성이다 | 세포 생존률 97% — ISO 10993-5 비독성 기준(70%) 상회 |
| 친환경이다 | 효소에 의한 생물학적 분해 경로가 알려져 있다 |
| 최고다, 탁월하다 | 측정 가능한 범위에서 최솟값이다 |
| 혁신적이다 | [구체적 기술 메커니즘으로 대체] |

---

### A-4. 카피 구조 템플릿

모든 텍스트 콘텐츠는 아래 4단 구조를 따른다.

```
1. 헤드라인 (선언형, 10–18자)
   수치 또는 핵심 사실로 시작
   
2. 서브라인 (근거, 수치 1–2개)
   헤드라인의 근거를 데이터로 즉시 제시
   
3. 바디 (맥락, 실제 의미)
   오해 소환 → 교정 → 메커니즘 → 의미 재정의
   
4. 클로징 (선언형)
   브랜드 포지셔닝 또는 철학으로 닫기
```

**카피 예시 (올바른 방식)**
- "클린은 선언이 아닙니다. 자극 유발 인자 51% 감소. 무균 시험 적합. 멜라누아는 수치로 신뢰를 설계합니다."
- "더하지 않고 남깁니다. 보존제, 분산제, 충전제를 제거했습니다. 성능이 유지될 때만 뺍니다."
- "97%다. 이것은 세포독성 시험에서 세포가 살아남은 비율이다. 비독성 판정 기준은 70%다."

---

### A-5. 헤드라인 금지 프레임

| 금지 표현 | 이유 |
|---|---|
| "Chemical Debt", "화학적 부채" | 과도한 대립 프레임 |
| "혁신적인 소재" | 구체성 없는 주장 |
| "피부가 원하는 것" | 의인화 과잉, 근거 없음 |
| "업계 최초" | 검증 불가 주장 |

## PART B. Visual Identity — "The Aesthetic of Reality"

> **"가장 정직한 것이 가장 아름답다."**

멜라누아의 비주얼은 CG나 가상의 이미지가 아닌, 실재하는 기술의 경이로움을 Fine Art Photography의 관점으로 드러낸다. 데이터와 물성이 숨김없이 드러나는 방식이 멜라누아의 신뢰 자산이며, 정직한 아름다움이다.

---

### B-1. Color System

| 색상 | 코드 | 의미 | 용도 |
|---|---|---|---|
| **Melanoir Absolute Black** | `#000000` | 모든 빛을 흡수하는 우주적 블랙. 생명의 원천. | 주색, 배경, 타이포 |
| **Oxygen White** | `#FFFFFF` | 멸균된 의료 환경과 산소를 상징하는 순수함 | 여백, 대비 |
| **Rocket Silver** | `#A9A9A9` | 정밀도와 기술적 중립성 | 보조 텍스트, 데이터 오버레이 |

**운영 원칙**: 고채도 색상 사용 금지. 브랜드 컬러 외 컬러 추가 시 Chief Architect 승인 필수.

---

### B-2. Photography Direction

**핵심 방향**
- 실재감 우선: 실험실, 소재 텍스처, 측정 순간 중심
- 하이 콘트라스트: Deep Black vs High Light
- 미니멀 프레임: 여백과 정밀함 강조

**Subject (촬영 대상)**
- 멜라닌 소재 텍스처 매크로
- 실험실 장면 (계측, 분석, 정제 과정)
- 피부 접촉 순간 (시술 현장, 소재 적용)

**Lighting**
- 단일 키 라이트 (single key light)
- 강한 섀도우 — 소재의 물성을 드러냄
- 과도한 보정 금지

**Style 레퍼런스**
- Fine art scientific photography
- Candid precision — 연출되지 않은 정밀함

**금지 비주얼**
- 과장된 CG 표현, 네온·고채도 톤
- 과학적 근거 없는 Before/After
- 경쟁사 연상 요소
- 감성적 라이프스타일 이미지 (브랜드 레이어 제외)

---

### B-3. Data Overlay 원칙

비주얼에 수치를 삽입할 때의 기준.

- 숫자 1–2개만 사용. 정보 과밀 금지.
- 작은 활자 + 정밀 그리드로 배치
- 예: `0.00` (피부 자극 지수), `97%` (세포 생존률)
- 수치 옆에 기준(ISO 표준 번호) 병기 권장

---

### B-4. Image Prompt Framework (AI 이미지 생성용)

외주업체 또는 내부 AI 이미지 생성 시 사용하는 프롬프트 기본 프레임.

```
Subject: [melanin material / lab scene / skin contact macro / precision measurement]
Style: fine art scientific photography, candid precision
Lighting: single key light, deep shadow, high contrast
Color: monochromatic black-white, minimal silver accent
Mood: factual, precise, understated
Negative: no CGI, no neon, no lifestyle, no before-after, no high saturation
```

### B-5. Typography System

멜라누아의 타이포그래피는 Apple의 SF Pro 시스템에서 영감을 받은 구조를 따른다.
명료함, 위계, 여백을 통해 데이터와 언어가 자연스럽게 드러나도록 설계한다.

#### 서체 구성

| 역할 | 서체 | 비고 |
|---|---|---|
| **Primary (한국어·영문 통합)** | **Pretendard** | SF Pro와 구조적으로 유사. 한국어·라틴 문자 동시 지원. SIL OFL 1.1 (무료 상업 사용). CDN 배포 가능. |
| **보조 영문 (브랜드 강조용)** | **Inter** | Google Fonts 제공. SIL OFL 1.1. 정밀한 라틴 문자 표현에 사용. |
| **크리에이티브 디스플레이 (지원 환경 한정)** | **페이퍼로지 (Paperlogy)** | 지원 플랫폼에서만 사용. 미지원 환경은 Pretendard로 자동 폴백. |

**Fallback Stack (CSS)**
```css
font-family: 'Pretendard', 'Paperlogy', -apple-system, BlinkMacSystemFont,
             'Segoe UI', 'Inter', sans-serif;
```

**CDN 로드 (웹)**
```html
<!-- Pretendard -->
<link rel="stylesheet" crossorigin
  href="https://cdn.jsdelivr.net/gh/orioncactus/pretendard@v1.3.9/dist/web/static/pretendard.css" />

<!-- Inter (선택) -->
<link href="https://fonts.googleapis.com/css2?family=Inter:wght@300;400;600;700;900&display=swap"
  rel="stylesheet">
```

#### Type Scale (Apple 구조 적용)

| 용도 | 크기 | 굵기 | Letter-spacing | 용례 |
|---|---|---|---|---|
| **Display / Hero** | 7.2rem | 900 | -0.05em | 제품 히어로 타이틀 |
| **Title 1** | 5.6rem | 700 | -0.03em | 섹션 주요 제목 |
| **Title 2** | 4.2rem | 700 | -0.02em | 서브 섹션 제목 |
| **Headline** | 3.5rem | 700 | -0.02em | 강조 문장, 통계 수치 |
| **Section Label** | 1.4rem | 600 | +0.15em (uppercase) | 섹션 카테고리 레이블 |
| **Body Large** | 1.65rem | 400 | -0.01em | 주요 본문, 제품 설명 |
| **Body** | 1.35rem | 400 | 0 | 일반 본문 |
| **Caption / Note** | 1.1–1.2rem | 400 | 0 | 법적 고지, 주석 |
| **Data / Stat** | 3.5rem | 700 | 0 | 수치 강조 표시 |

#### 타이포그래피 운칙

- Line-height: 본문 1.8 / 제목 1.08–1.25
- 한국어는 `word-break: keep-all` 적용
- 수치(Data/Stat)는 색상 강조 가능 — 본문과 대비되는 브랜드 컬러 또는 black
- 제목에 고채도 컬러 사용 금지

### B-6. Logo System

멜라누아 로고는 **Symbol**과 **Wordmark**로 구성된다.

| `MELANOIR_logo.ai` | 원본 벡터 | — | 인쇄, 수정 작업 |

#### 로고 사용 원칙

- **기본형**: sg1(수직) 또는 sg2(수평) — 공간에 따라 선택
- **심볼 단독**: 아이콘, 파비콘, 소형 응용에서만
- **워드마크 단독**: 이미 심볼이 노출된 레이아웃에서 중복 제거 시
- **색상**: Black(#000000) 또는 White(#FFFFFF) — 그레이 변형 금지
- **최소 여백(clear space)**: 심볼 높이의 0.5배 이상 사방 확보
- **변형 금지**: 비율 변경, 회전, 색상 변경, 효과 추가 금지
- **원본 파일**: 인쇄·편집 작업은 반드시 `MELANOIR_logo.ai` 사용
